# Contributing Guide

Thank you for considering contributing to Devnovate Blogs!

## How to Contribute
1. **Fork the repository** and create your branch from `main`.
2. **Write clear, modular code** following SOLID and Clean Code principles.
3. **Add or update tests** for your changes in `tests/`.
4. **Document your changes** in code comments and, if needed, in `docs/`.
5. **Run linting and tests** before submitting a pull request.
6. **Submit a pull request** with a clear description of your changes.

## Code Style
- Use Prettier and ESLint for formatting and linting.
- Follow naming conventions and keep functions small and focused.

## Security
- Never commit secrets or credentials.
- Validate and sanitize all input.

## Communication
- Use issues for bugs and feature requests.
- Be respectful and constructive in all discussions.
